Training Pipeline Roadmap

  • 1. Frame Extraction

    Cameras record at 30Hz, while IMU sensors record down to 50Hz. Python extracts precise RGB frames strictly tied to hardware UNIX timestamps.

  • 2. Automated Annotation (YOLO + CLIP)

    100% automated offline pipeline. YOLO-World generates spatial bounding boxes from a custom vocabulary, before PIPING crops directly to CLIP for zero-shot text description profiling.

  • 3. Manual Feature Refinement

    Scrub through the video dataset and manually draw bounding boxes through the interactive Canvas UI for the rarest edge-cases or missed items.

  • 4. High-Frequency Synchronization

    The backend matches disparate time-domains (33ms video vs 20ms physics) using Pandas merge_asof, applies SciPy Butterworth DSP filters, and drops anomalous gap data.

  • 5. Core PyTorch Network Training

    Trains the LateFusionNetwork on processed datasets natively inside the application. Live JSON stdout telemetry feeds the real-time Dashboard Chart.js graphs.

  • 6. Real-Time Inference

    Simulate predictions seamlessly utilizing PyTorch evaluation modes over unseen synchronized sensor feeds.

State-of-the-Art (SOTA) Analysis

A peer-reviewed architectural derivation highlighting critical divergences between the CycleSafe Multimodal framework and contemporary baseline methodologies within micro-mobility surface classification literature.

1. Resolution of the "Visual-Blindness" Paradigm

THE SOTA FLAW Contemporary road classification models heavily depend on active Computer Vision pipelines (e.g., YOLO/CNN configurations mounted on handlebars). These fail precipitously under sub-optimal parameters: twilight operations, lens occlusions from mud, or solar glare. Continuous real-time visual processing also incurs high computational overhead, degrading battery endurance.

CycleSafe Methodology: We execute an architectural inversion. Computer Vision (CLIP/App-Based Annotator) is strictly relegated to offline preprocessing to automate ground truth assignment. Real-time inference executes "blindly" via a `PhysicsTransformer` analyzing purely 1-Dimensional IMU telemetry, achieving zero-degradation performance during total visual occlusion.
